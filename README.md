# L.O.V.E. — Relationship Support Agent
**Listen. Open Dialogue. Validate Feelings. Encourage Solutions.**
 
RSM 8430 — Applications of Large Language Models | Group 18

## What This Is
 
A conversational support agent that helps users reflect on relationship conflicts and generate constructive communication strategies. The agent follows the **L.O.V.E. framework** — it listens first, asks follow-up questions to understand the situation, validates the user's feelings, and then encourages grounded solutions drawn from real therapist-authored examples.

This is **not a therapy bot**. It is a supportive companion that retrieves and synthesizes guidance from a curated knowledge base, helping users prepare for difficult conversations and reflect on relationship dynamics.

## Setup & Running Instructions
 
### Prerequisites

- **Python 3.9+**
- A running LLM endpoint (course-provided Qwen or any OpenAI-compatible API)

### 1. Clone the repo

```bash
git clone <repo-url>
cd RSM8430-Group18
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

### 3. Build the vector index (run once)

This embeds the 225 curated CounselChat Q&A pairs into ChromaDB:

```bash
python rag/build_index.py
```

You should see `✅ All 225 documents indexed successfully.`

### 4. Configure the LLM endpoint

Set these environment variables to point to your LLM:

```bash
export LLM_API_BASE="https://rsm-8430-finalproject.bjlkeng.io"   # course-provided endpoint URL
export LLM_API_KEY="your-api-key"                      # student number
export LLM_MODEL="qwen3-30b-a3b-fp8"                  # model name
```

### 5. Run the app

```bash
streamlit run app/main.py
```

Open `http://localhost:8501` in your browser. You should see the L.O.V.E. welcome screen.

## L.O.V.E. Conversational Framework

Every response follows this structure:

| Step | Name | What the agent does |
|------|------|---------------------|
| **L** | Listen | Reflects back what the user said to show understanding |
| **O** | Open Dialogue | Asks 1–2 follow-up questions to explore the situation |
| **V** | Validate Feelings | Names the user's emotions and normalises the experience |
| **E** | Encourage Solutions | Offers 2–3 practical suggestions grounded in therapist examples |

The agent intentionally **does not jump straight to advice**. It prioritises making the user feel heard first.

## Knowledge Base
 
**Source:** [nbertagnolli/counsel-chat](https://huggingface.co/datasets/nbertagnolli/counsel-chat) via HuggingFace
 
**What it is:** Real therapist-authored Q&A pairs covering relationship conflict, trust, communication, breakups, and marriage.
 
**What we use:** 225 curated Q&A pairs filtered from ~2,780 raw rows. Filtering criteria:
- Must mention a romantic partner as the core subject
- Answers must be substantive (80+ characters)
- Hard exclusions: crisis content (self-harm, suicide, abuse)
- Two-tier system: Tier 1 (113 high-confidence rows) + Tier 2 (112 extended coverage rows)
 
Each Q&A pair is one document unit embedded as-is — no arbitrary chunking.

## Agent Capabilities
 
### Intent Routing
Classifies every user message into one of:
`rag_qa` | `build_plan` | `reflection` | `save_plan` | `retrieve_plan` | `unsafe` | `out_of_scope`

Uses keyword rules for high-confidence matches, with LLM fallback for ambiguous inputs. Respects in-progress multi-turn actions.
 
### Knowledge Q&A (RAG)
Retrieves top-3 similar therapist responses from the knowledge base and synthesizes a grounded answer with source attribution.
 
### Actions
 
| Action | Type | Description |
|--------|------|-------------|
| Build Conversation Plan | Multi-turn | Collects issue, goal, tone → generates opening statement, talking points, boundary phrase |
| Reflection Exercise | Single-turn | Generates guided self-reflection prompts based on the user's situation |
| Save / Retrieve Plan | Single-turn | Persists generated plans by session ID for later retrieval |
 
### Guardrails
- Crisis/self-harm detection → redirects to professional resources
- Abuse/violence signals → safe fallback response
- Medical/legal advice requests → out-of-scope rejection
- Prompt injection detection
- Disclaimer: agent is not a licensed therapist
 
### Session Persistence
State is stored in SQLite, including conversation history, filled action slots, and saved plans. State persists across app restarts and can be retrieved by session ID.

## Known Limitations
 
- Knowledge base is limited to 225 documents — niche relationship scenarios may not retrieve strong matches
- Agent does not provide professional mental health advice and is not a substitute for therapy
- Session state is local only — no multi-device sync
- LLM responses are grounded in retrieved content but may still reflect biases present in the CounselChat dataset
